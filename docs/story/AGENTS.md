# 小说写作规则

本小说的正式书名为《我与篮球的距离》。

凡涉及本项目小说的续写、补章、扩写、改写或润色，必须遵守以下规则：

1. 每次开始续写、补章、扩写、改写、润色或其他正文修改前，都必须完整阅读全部框架文档，包括 `WRITING_GUIDE.md`、`STORY_BIBLE.md`、`OUTLINE.md`、`characters/README.md`、三位核心人物档案及郑天锐的人物档案；同时完整阅读原稿 `content/1.txt`、`content/6.txt`、`content/7.txt`，并以这些原稿为语言风格的最高依据。
2. `OUTLINE.md` 用于人物与剧情框架；后补的第 2—5 章只能作为连续性参考，不能取代原稿成为风格样本。
3. 故事主体固定在 2014 年的深圳光明新区公明中学。今天的马田街道当时尚未成立，正文采用 2014 年的行政称谓。
4. 主要学生角色统一为初三年级；已明确的背景和班级设定以 `STORY_BIBLE.md` 为准。
5. 新增真实道路、学校、建筑、公交、商场、公园或其他地方信息前，必须查询其在目标时间是否存在及当时名称。优先使用政府、学校和交通等官方来源。
6. 保留原稿的少年热血、比赛速度、口语玩笑和直接情绪，但不要故意复制错别字、乱码或规则错误。
7. 使用 2014 年能够成立的语言、产品、网络文化、NBA 信息和交通条件，不得混入后来的地铁站、短视频梗或新建筑。
8. 每次续写、补章、扩写、改写、润色或其他正文修改完成后，都必须检查本次内容是否新增、删除或改变了人物、关系、班级、家庭、地点、比赛结果、时间线、剧情安排、写作规范或其他既有设定，并同步更新所有受影响的框架文档，包括但不限于 `STORY_BIBLE.md`、`OUTLINE.md`、`WRITING_GUIDE.md`、`characters/README.md` 及 `characters/` 中对应的人物档案；不得只修改正文而遗漏框架文档。
9. 每次正文创作必须依次执行以下四步，不得跳步：
   1. **读取整个框架**：完整阅读第 1 条规定的全部框架、人物档案和原稿风格样本，并阅读承接本次剧情所需的相邻章节。
   2. **列出大纲**：先向用户提交本次创作的详细大纲，写清核心人物、矛盾、场景顺序、关键回合、结尾钩子及不能改变的设定；用户确认前不开始正式正文。
   3. **写文章**：按照确认后的大纲完成正文，并依照第 8 条同步更新受影响的框架文档。
   4. **检查刚写内容的逻辑**：正文完成后，单独复查时间线、前后章衔接、人物动机、对话指代、比赛比分、得分变化、篮球动作、场景移动、物品来源及结尾伏笔；发现问题先修正，再交付最终版本。
